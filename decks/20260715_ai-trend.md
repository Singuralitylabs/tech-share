## 付き合い方が"設計"に変わった

1. **Prompt**
   *2020〜22年*
   指示を工夫する
2. **Context**
   *2025年6月〜*
   渡す文脈を設計する
3. **Harness**
   *2026年2月〜*
   ツール・状態・安全レールを整える
4. **Loop**
   *発想は2023年・実用は2026年*
   自律的に回り続ける仕組みへ

**AutoGPT（2023年）が続かなかったのは、②と③が無かったから。** 発想だけ3年早く、土台がいま揃った。

※ 4段の並びは整理のためのフレーム。各語自体は実際に使われている。

## 道具 → 自分で回り続ける相棒

> 同じモデルでも、**"職場"が違えば別人**になる。

- **ループ**＝考える → 動く → 結果を見る → また考える
- **ハーネス**＝その職場。**ツールと権限 ／ 記憶の出し入れ ／ 仕事の分担 ／ 止め方**
- Claude Code や Codex が売っているのは、モデルではなく **"職場"ごと**
- 人の仕事は「うまく指示する」から **「職場を設計する」** へ

<!--
前スライドで出した Harness / Loop の中身を、たとえを崩さずに具体化する枚。
ハーネス＝モデルを包む実行環境（ツール実行・ファイルアクセス・サブエージェント委譲・コンテキスト管理）。
ループ＝ReAct（2022）由来の 考える→動く→観察→また考える。
「同じモデルでも職場が違えば別人」が一番効く。Claude CodeとCursorで同じモデルを使っても出来が違う理由。
ハーネスの仕事の半分は"止めること"（無限ループ・コンテキスト溢れ・費用の暴走を防ぐ）。聞かれたら答える。
次パート（新サービスが溢れる）への「なぜ」の前振りでもある：職場ごと売る商売が増えた、という流れ。

## 規制が出そろい、日本は昨日動いた

- **AI法の施行が、この半年に集中した**
  - 台湾 1/14 ／ 韓国 1/22 ／ 米カリフォルニア・テキサス 1/1。**EU は 3週間後の 8/2** に執行開始
- **日本の AI推進法には罰則がない** — 指導・勧告・公表まで。推進を優先した設計
- **7/14、政府がAI基本計画を改定**（閣議決定は昨年12月）— **わずか半年での改定**
  - 理由は「**クロード・ミュトス**の登場で、サイバー攻撃の脅威が顕在化した」
  - AI法を含む制度の見直し／重要インフラ対策／政府システムの緊急点検／「**AI主権**」

**傾向：規制は各国バラバラ。日本は追随側で、前提が半年で書き換わる。**
