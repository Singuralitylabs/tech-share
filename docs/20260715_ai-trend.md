# 生成AIトレンド紹介 プレゼン骨子（会員向け・約18〜20分）

> **狙い**：幅広い会員層（エンジニア〜非エンジニア混在）に、この半年の生成AI動向を「知識アップデート」として持ち帰ってもらう。
> **通しのメッセージ**：性能はもう横並び。勝負の軸は「コスト」「使い方（自律化）」「distribution・統合」「安全・規制」へ移った。1強の時代から"用途別ベスト"の時代へ。
> **各パートの型**：できるだけ「半年前はこうだった → 今はこう」のビフォー/アフターで統一すると、アップデート目的に刺さる。
> **数字の扱い**：混在層向けなので、1パートにつき"覚えて帰る数字/エピソード"は1つに絞る。細かいベンチ数字は出さない。

## 1. オープニング：半年で景色が変わった（2分）

**話すポイント**
- この半年、生成AIは「毎月のように景色が変わる」フェーズに入った。
- 象徴的な事実：**ChatGPTの市場シェアが2026年3月、初めて50%を割った**。かつて一強だったのが、今や"多極化"へ。
- 今日は「何が出たか」を並べるのではなく、**なぜこう変わったのか**を4つの軸で見ていく。

**見せる数字/エピソード（1つ）**
- ChatGPTシェア：ピーク時ほぼ独占 → 3月に50%割れ → 5月末 約46%（Sensor Tower / True Audience）。
  ※ Similarwebのウェブ訪問ベースだと約54%。**「測り方で数字が変わる」ことだけ軽く断っておく**と誠実。

**ひとことスライド文言案**
> 「AIの主役が、1つから"用途別"へ」

## 2. モデルが出すぎて追えない（3分）

**話すポイント**
- 主要3社だけでも、この半年でこれだけ出た。**紹介というより"ペースの速さ"を体感してもらう**。
- Anthropic：Opus 4.7（4月）→ Opus 4.8（5月）→ Fable 5（6月）。**Opus 4.7→4.8はわずか約42日**。
- OpenAI：GPT-5.5（4月）→ GPT-5.6（Sol/Terra/Luna、6〜7月）。
- SpaceXAI（旧xAI）：Grok 4.3 → Grok 4.5（7月）。
- 加えて中国勢（GLM系・DeepSeek系など）も台頭。※個別バージョン名には深入りしない。

**見せる数字/エピソード（1つ）**
- 「**Anthropicは約6週間で次を出す**」。半年前のモデルはもう最新ではない、という感覚。

**ひとことスライド文言案**
> 「半年で、ほぼ全部が入れ替わった」

**補足メモ**：時系列を1枚の帯（タイムライン）で見せると"速さ"が視覚的に伝わる。個々のスペックは語らない。

## 3. 性能は横並び、勝負はコストへ（3分）

**話すポイント**
- ここが今回の一番の"認識アップデート"。**トップ勢の性能はもう僅差に収束した**。
- 難関ベンチでもトップ4が数ポイント差に収まる。＝「どれが賢いか」で選ぶ時代は終わりつつある。
- だから競争の主戦場は**コストとトークン効率**へ移った。
  - Grok 4.5：$2 / $6（100万トークン、入力/出力）と低価格を前面に。「Opus級だが速く安い」という打ち出し。
  - OpenAI：Terra（GPT-5.5同等を約半額）、Luna（最安・最速）とティアを分けてコスト最適化。
  - Anthropic：Opus 4.8に fast mode を追加（高速版を従来より安く）。

**見せる数字/エピソード（1つ）**
- 価格比較を1枚で。例：Opus 4.8 $5/$25 vs Grok 4.5 $2/$6 vs GPT-5.6 Luna $1/$6。
  「**同じような性能でも、値段はこれだけ違う**」を体感させる。

**ひとことスライド文言案**
> 「もう"賢さ"では差がつかない。差は"コスト"に」

**補足メモ**：混在層向けなので、ベンチの数字そのものは出さず「横並びになった」という結論だけ。価格は誰にでも刺さるので、ここは具体数字OK。

## 4. 付き合い方が変わった：Prompt → Context → Harness → Loop（3分）

**話すポイント**
- モデルの話から、**"使い方"の話**へ。この半年で、AIとの付き合い方の考え方自体が進化した。
- ざっくり言うと、AIは「1回きりの道具」から「**自分で回り続ける相棒**」へ変わった。
- 進化の4段階（※これは業界標準用語というより整理のフレーム、と断っておく）：
  1. **Prompt**（指示を工夫する）
  2. **Context**（渡す文脈を設計する）
  3. **Harness**（ツール・状態・安全レールでAIの"作業環境"を作る）＝ agent = model + harness
  4. **Loop**（スケジュール実行・自己検証・停止条件まで含めた"自律ループ"を設計する）
- ポイントは「**人間が毎回プロンプトを打つ**」から「**回り続ける仕組みを設計する**」への転換。

**見せる数字/エピソード（1つ）**
- 一言のたとえ：「**プロンプトを書く時代から、AIが働く"職場"を用意する時代へ**」。

**ひとことスライド文言案**
> 「道具 → 自分で回り続ける相棒」

**補足メモ**：非エンジニア層が主役のパート。専門用語は最小限、たとえで押す。次パート（新サービス）への"なぜ"の前振りになる。

## 5. だから新サービスが溢れている（4分）

**話すポイント**
- 「AIが自律的に回るようになった（前パート）→ 気づけば"モデルを使うためのサービス"が一気に増えた」。
- **数の多さそのものがメッセージ**。1つずつ深掘りせず、リストで畳みかける。
- カテゴリで整理すると迷子にならない：
  - **デザイン・制作**：Claude Design（テキストからデザイン/プロトタイプ/スライド）、Claude in PowerPoint / Excel
  - **コーディング・開発**：Claude Code、Grok Build、OpenAI Codex（ChatGPTアプリに統合）
  - **知識ワーク・エージェント**：Claude Cowork、GPT-5.6 の Ultra モード（並列サブエージェント）
  - **ブラウザ・実行**：Claude in Chrome、各種 MCP 連携
- 半年前は「モデルそのもの」の話が中心だった。今は「**モデルを"使う"レイヤー**」が主戦場に。

**見せる数字/エピソード（1つ）**
- Claude Design を代表例に一言だけ：「**文章で指示するだけで、デザインやスライドの下地ができる**」。
  （深掘りはせず"こんなものまで出た"の象徴として）

**ひとことスライド文言案**
> 「主役は"モデル"から"モデルの使い方"へ」

**補足メモ**：サービス名がずらっと並ぶスライド1枚で"量"を見せるのが今回の狙い。読み上げは代表2〜3個で十分。

## 6. 業界が動いた：買収と統合（3分）

**話すポイント**
- モデル・サービスの裏で、**業界の地図そのものが描き変わった**。単なるゴシップではなく「垂直統合」の動き。
- **SpaceXがxAIを買収 → 統合主体は "SpaceXAI"**（2026年2月）。さらに**AIコーディングツール Cursor を約600億ドルで買収する計画**（6月開示）。→ Grok 4.5 は Cursor の実開発データで訓練。**compute × 開発ツール × モデルラボの一体化**。
- **Apple × Google**：新しい Siri が Gemini で動く（約14億台のiPhone）。→ Geminiの急成長は"性能"より"**どこに載っているか（distribution）**"が主因。
- 一方 Anthropic（Claude）は、消費者シェアは1桁台でも**企業・開発者で強い**（対OpenAIの新規案件で高勝率、有料転換率も最高水準）。

**見せる数字/エピソード（1つ）**
- 「**Cursor買収 約600億ドル**」または「**Geminiが14億台のiPhoneのSiriに**」。スケール感を1つ。

**ひとことスライド文言案**
> 「勝負を分けるのは、性能より"どこに載るか"」

## 7. 安全と規制が現実の制約に（3分）

**話すポイント**
- 上半期の"目玉"。**AIの能力が、安全対策の整備を追い越し始めた**。
- **Anthropic の Mythos Preview / Project Glasswing**（4月）：一般公開しないほど強力なモデルが、主要OS・ブラウザで**数千件のゼロデイ脆弱性を自律発見**（27年間見つからなかったバグまで）。あまりの反響に政府・大手銀行が緊急協議。
  - 対応として、危険な能力は**限定パートナーにゲート**し、安全レールを下位モデル（Opus 4.7）で先に検証してから広げる、という運用に。
- **GPT-5.6 は米政府審査を経て段階公開**（6月プレビュー → 7月GA）。frontier モデルの公開が"事実上ワシントンを通る"時代に。
- **Fable 5 / Mythos 5 は輸出規制で一時停止**（6/12停止 → 7/1復旧）。安全・地政学がプロダクトの提供可否に直結。

**見せる数字/エピソード（1つ）**
- 「**公開しないほど強いモデルが、27年間誰も気づかなかったバグを自力で見つけた**」。危機感と可能性の両面を象徴。

**ひとことスライド文言案**
> 「能力が、安全対策を追い越し始めた」

**補足メモ**：怖い話で終わらせず、「だから規制・審査が現実の制約になった」という構造の話まで運ぶ。

## 8. クロージング：用途別ベストの時代（1.5分）

**話すポイント**
- この半年の要点を4つで振り返る：
  1. 性能は横並び → 差は**コスト**へ
  2. AIは**自律的に回る相棒**へ（使い方が主役）
  3. 勝敗は**distribution・統合**が握る
  4. **安全・規制**が現実の制約に
- 結論：「1強」の時代は終わり、**用途で使い分ける時代**へ。
- （知識アップデートが目的なので）行動を強く迫るより、「**今、こういう地図になっています**」と現在地を確認して締める。
- 締めの一言例：「**"どれが一番か"より、"何にどれを使うか"**」。

**ひとことスライド文言案**
> 「1強の時代から、用途別ベストの時代へ」

## 付録：削ってよい/注意する要素

- **詳細なベンチマーク数字**：混在層では逆効果。「横並びになった」の結論だけでよい。
- **中国モデルの個別バージョン名**（GLM 5.2 等）：裏取りが難しく、幅広い層には細かすぎる。「中国勢も台頭」で十分。
- **数値の出典**：市場シェア等は調査会社・測定手法で差が出る。1箇所だけ「測り方で変わる」と断れば、以降は繰り返さなくてよい。
- **時間が押したら**：パート2（モデル羅列）とパート3（コスト）は圧縮しやすい。パート4〜5（使い方・新サービス）と7（安全）は今回の主役なので残す。

### 配分の考え方
- 各パート2枚に揃えて**リズムを一定**にする。区切りスライド（各パート冒頭の見出しページ）は今回は設けず、各パート1枚目の見出しで代替。
- 密度が高いのは**パート5（新サービスを"数"で見せる）**。詰まる場合は2枚に割ってよい。
- **パート2（モデル羅列）**はタイムライン1枚に集約可能。時間・枚数を削るならここが最初の候補。

### Claude Codeへの実装メモ（Marp）
- **ツール**：Marp（Markdown → スライド）。`marp-cli` でHTML/PDF/PPTX出力を想定。フロントマターで `marp: true`、`theme`、`paginate: true` を指定。
- **スライド区切り**：`---` で区切る。各スライドは上表の「各スライドの中身」に対応させる。
- **本文の元ネタ**：各パートの「話すポイント／見せる数字・エピソード／ひとことスライド文言案」を流用する。スライド上は文字を詰め込みすぎず、**ひとことスライド文言案を見出し**に、要点は3〜4項目の箇条書きに圧縮する。
- **図版が要るスライド**：①ChatGPTシェア推移（折れ線）②リリースのタイムライン帯 ③価格比較表 ④Prompt→Loopの4段階図 ⑤カテゴリ別リスト。まずはMarpの表・箇条書き・簡単なCSSで表現し、凝った図は後追いで差し替える方針。
- **デザイン方針**：テンプレ然としない配色・字組みを意識（詳細は別途デザイン指定があれば従う）。1枚1メッセージを徹底。
- **数値の出典注記**：市場シェア等は測定手法で差が出るため、該当スライドかフッター、または最終「参考」スライドに出典と測定基準を明記する。
